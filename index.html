Paper Year Input Output Pro Con
MatLat CVPR 2026 Given 3D mesh + material / appearance condition PBR texture / material channels A close match for mesh-based material texturing. It works on PBR channels directly, with the material latent space, VAE fine-tuning, and patch locality regularization all aimed at cleaner material details. It does not try to repair mesh or UV issues. Fine-grained semantic regions or part-level controls would still need to come from another module.
LumiTex ICLR 2026 Reference image + mesh. The official inference format uses ref.png plus an OBJ/GLB mesh; the method also conditions on mesh normals, canonical coordinates, multiview renders, and DINOv2 / VAE image features. PBR texture maps: albedo / basecolor, metallic, and roughness, with geometry-guided LVSM texture inpainting to complete occluded or missing regions for relightable materials. Targets PBR material decomposition instead of a single attractive RGB texture, using illumination context to avoid baking highlights and shadows into albedo. The multi-branch albedo and metallic-roughness design, lighting-aware material attention, and view-space / LVSM inpainting are especially relevant for high-fidelity mesh + image texturing. Training is expensive: the paper reports roughly 92K objects, 30 views, 3 environment maps, and about 106 GPU days. It is aimed at whole-mesh PBR texturing, not multipart per-part alignment, part-boundary consistency, or repeated-part consistency.
MV2UV 2026 Untextured mesh + known UV + image / multiview prompts High-quality UV texture map Directly targets the "put a texture on this mesh" setting. The mix of multiview priors, UV refinement, and inpainting is useful for occluded, unseen, and view-conflicting regions. Mesh and UV quality still matter. The output is mainly an RGB UV texture map, not a full PBR material breakdown.
UniTEX CVPR 2026 3D shape / mesh + prompt or multiview texture synthesis prior Consistent 3D texture in unified 3D functional space Uses a Large Texturing Model with Texture Functions, which sidesteps some of the topology ambiguity of UV-based methods. It is most interesting when consistency and coverage matter more than a simple one-shot texture map. The two-stage setup is heavier than direct UV generation. If the target is a standard texture file, export and baking details still need to be checked.
CustomTex CVPR 2026 Untextured indoor 3D scene + reference images for each object instance Unified high-resolution scene texture map Useful when the asset is an indoor scene rather than a single object. Instance-level reference control helps each object follow its own reference while reducing baked-in shading and artifacts. Less relevant for generic single-object mesh texturing. It also assumes per-instance references and a more optimization / distillation-style workflow.
NI-Tex CVPR 2026 Garment mesh + garment image / non-isometric deformation reference Spatially aligned PBR garment materials A strong fit for garments, especially when non-isometric deformation makes direct texture transfer hard. The output is spatially aligned PBR material suitable for a production garment pipeline. The scope is narrow. It should not be treated as a general solution for hard-surface objects, props, or indoor scenes.
Hitem3D 2.0 2026 Single or multiple reference images + 3D geometry / generated multiview conditions High-fidelity, geometry-aligned 3D texture Combines 2D multiview priors with a native 3D representation. The pipeline pays attention to geometric alignment, view consistency, illumination uniformity, and unseen areas. It is a fairly heavy pipeline. For a clean mesh with good UVs, this may be more machinery than the task needs.
EmissionGen 2026 Untextured 3D mesh + emission material reference / prompt Emission texture / emissive material maps Fills a specific gap: emissive texture generation for LEDs, signs, cyberpunk-style assets, and similar materials. Most texture methods still spend more attention on albedo / metallic / roughness. The scope is intentionally vertical, closer to a benchmark and baseline than a general PBR material generator.
GarmentPainter 2026 Garment mesh UV position map + character reference image / type selection Garment UV texture A straightforward garment UV texturing route. The UV position map gives structural guidance, and component-level generation is useful for clothing parts. It is built around garment assumptions: UV position maps, clothing components, and garment priors. Arbitrary meshes are outside its comfort zone.
NaTex 2025 3D geometry / mesh condition Dense color point cloud / surface texture in 3D space Predicts texture color in 3D space instead of stitching together 2D views. That makes occlusion, mesh-texture alignment, and cross-view consistency less brittle. The representation is 3D-native. A standard game / DCC pipeline would likely need a baking step before it becomes a UV texture asset.
CaliTex 2025 Mesh / geometry condition + appearance prompt or multiview condition Seamless, view-consistent texture The main appeal is its geometry-calibrated attention: the texture model is asked to pay attention in a way that follows the 3D structure. Part-Aligned Attention and Condition-Routed Attention are aimed at view mismatch and ambiguous attention. The extra calibration and two-stage DiT design make it harder to drop into a lightweight UV inpainting pipeline. It also puts more pressure on the part / geometry conditions.
LaFiTe 2025 Arbitrary mesh / surface + condition input 3D-native sparse latent color field / surface texture Moves the texture representation away from fixed UVs or rendered views and learns a sparse latent color field on the surface. That is a good fit for seamless 3D-surface texture generation, editing, and material synthesis. It is not a conventional UV texture-map output. For production assets, the color field would usually need to be converted or baked.
TEXTRIX 2025 3D model / mesh + condition input Latent 3D attribute grid for texture synthesis and 3D part segmentation Puts color and other attributes into a volumetric 3D latent grid. This avoids some multiview fusion artifacts and also gives the method a path toward fine 3D part segmentation. It is broader than texture generation alone. It may also be sensitive to poor mesh quality, topology defects, and self-intersections.
MaterialMVP 2025 3D mesh + image prompt PBR texture maps: albedo, metallic, roughness Good at separating color from roughness/metallic instead of baking lighting into one texture Needs a clean mesh/UV, and it does not decide where object parts should be
Hunyuan3D 2.1 2025 Single image Textured 3D asset with mesh and PBR textures Handy when there is only one photo and we need a complete textured asset It regenerates the object, so small parts and topology can drift
Meshy N/A Text, image, or uploaded mesh Export-ready textured 3D model or retextured mesh Fast and convenient for getting a previewable textured model Closed service, so exact behavior and part preservation are hard to inspect
TRELLIS 2024 Text or image prompt 3D asset decoded as mesh, 3D Gaussian, or radiance field Can produce several 3D representations from the same prompt or image More about generating a new asset than keeping a given multipart mesh unchanged
Paint3D 2024 Untextured mesh + text or image prompt Lighting-less 2K UV texture map Removes much of the lighting baked into view-based textures Mainly outputs RGB texture and still depends on the given UV/mesh quality
TEXGen 2024 Mesh + text, single image, or sparse views High-resolution UV texture map Works in UV space, which helps with completion and seam handling Does not really reason about materials or object parts beyond the texture map
MVPaint 2025 3D mesh + text prompt High-resolution seamless UV texture Synchronized views make the painted texture more consistent around the object Still a multi-stage RGB texture pipeline, without geometry or PBR decomposition
TexPainter 2024 3D mesh + text prompt Multi-view-consistent mesh texture Color fusion helps when different rendered views disagree Per-object optimization costs time and does not add material maps
SyncMVD 2024 3D mesh + text prompt Consistent UV texture for the input object Shares the texture during denoising, so different views fight less Fixed camera views can still miss hidden or thin parts
TexGen 2024 3D mesh + text prompt RGB UV texture map Sampling and resampling across views helps reduce seams Can still paint details that do not quite fit the geometry
Text2Tex 2023 3D mesh + text prompt Textured mesh / UV texture Simple progressive painting strategy; easy to understand and compare against Sequential painting can accumulate seams and view-to-view mismatch
TEXTure 2023 3D shape + text, image, or scribble guidance Generated, edited, or transferred texture Covers texture generation, editing, and transfer in one framework Iterative view painting may bake lighting and inconsistent details
MVPainter 2025 Reference image + known 3D geometry PBR textured mesh Uses geometry control to keep generated views closer to the reference Needs a good reference and has more moving parts in the pipeline
Material Anything 2025 Textured or texture-less 3D object UV-ready PBR material maps Useful for turning an existing object into cleaner PBR material maps Material-focused; it does not solve part segmentation or geometry layout
MeshGen 2025 Single image PBR textured 3D mesh Builds both shape and PBR texture from one image Can change the target topology, so it is not ideal for preserving multipart meshes
